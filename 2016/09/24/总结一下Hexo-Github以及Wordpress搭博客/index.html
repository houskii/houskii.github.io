用Hexo/Wordpress搭博客总结

目前对于非前端来说，搭Blog的主流方式是用网上现成的框架，主流的有Hexo以及Wordpress，刚好这周我把这两个框架都简单尝试了一下，在此做一下小小的总结

Wordpress

后端使用php，有后台管理，文章存在数据库里。要搭Wordpress首先要自己的服务器，这里我用的是腾讯云，系统为CentOS 64位。

先配置httpd/php/mysql/lrzsz的环境，然后启动这3个服务

用yum安装时由于历史原因（mysql被oracle收购后有闭源的风险），所以install mysql是装不了的……换成install mariaDB，但还是可以通过mysql来访问

然后访问mysql并创建账号以及一个数据库

接着到workpress下载tar.gz的包，然后用shell cd 到/var/www/html/wordpress目录中，用rz或者ftp5上传这个压缩包，接着用tar -zxvf [filename]解压这个包

输入vi wp-config-sample.php，修改以下信息并rename为wp-config.php

这时候访问http://服务器公网ip/wordpress进行设置就Ok了

——————————————以上是基本设置——————————————————

在设置完wordpress基本属性后，你会发现默认theme并不好看，可以在网上下载主题然后扔到/var/www/html/wordpress/wp-themes/文件夹下，就可以在wp的管理界面下找到这个主题。

但是当你想要下载插件的时候，你会被要求输入主机ip/账户/密码，当时我输了好久都给我提示不对，最后才发现对插件的设置是要求启用ftp的……于是还要去给服务器装vsftpd，这玩意的设置可以自行google……账号密码也是ftp的账号密码

但是输入以后还可能装不了插件，找了半天log发现是权限不够的问题，本着懒的心态，可以直接sudo chmod -R 777 /var/www/html/wordpress给所有用户组读写执行的权限……

这里其实对wordpress的设置就差不多了，但是我还是不太满意，因为这玩意发表文章实在是太麻烦了，必须要登陆后台管理界面去写，虽然可以用邮件但是我测试了半天并不能成功……这时候寝室前端大神跑过来：“辣鸡框架后端还用php” _(:з」∠)__ 看了一下确实很捉急，我这个写Android的看了半天别人theme的代码，想改都不知道怎么改……

Hexo

大神推荐的框架，后端算是Node.js。网上的教程蛮多的，大多都是和Github page一起用的

先安装node.js，然后npm install hexo-cli、npm install hexo，基本上就搞定了，运行hexo init直接在当前文件夹下创建一个hexo的基本页，然后输入hexo g或hexo generate静态化页面，hexo s或者hexo server启动本地服务器就可以在localhost:4000看到了，绑定了github以后通过hexo d或者hexo deploy把页面发到github上就完成了。

页面/主题等设置通过_config.xml文件修改，文章通过hexo new “[filename]”来创建，默认为md格式

说一下碰到的几个坑

  • hexo指令都要在init后的文件夹里执行，不然是没效果的
  • 每次执行都报错Error: Cannot find module './build/Release/DTraceProviderBindings'...，其实这玩意可以不理，当然网上也就解决办法，比如uninstall后再installhexo-cli和hexo，以及installhexp的时候加上--optional--no，然而我试过了都没效果。最后通过npm uninstall dtrace-provider这个组件解决（但是编译的时候依然有错）
  • 执行hexo g之前最好执行一次hexo clean清除一下上次编译留下来的数据，但是hexo clean会直接删除目录下编译成功的public文件夹。如果涉及到个人域名解析，需要在工程中添加CNAME文件，而hexo d是默认上传public文件夹内所有内容的，所以每次clean之后记得把CNAME放进去

域名解析

如果购买了腾讯云的学生套餐是会送一个域名的，想把这个域名和github pages绑定的话，首先要在yourname.github.io这个项目下创建一个CNAME文件，文件里填写你自己的域名（比如我填了houskii.cn），这时候访问yourname.github.io就会自动转跳到你填的地址

此时还要记得设置域名的解析，添加一个A地址，ip输入151.101.16.133（151.101.16.134），开启解析就OK了

总结

总结一下，Hexo好处在于天然支持md，纯前端，可以在本地server测试了以后再发布，缺点就是页面是静态的，也就是说哪怕只改一个字也要走generate->deploy这个流程

Wordpress好处在于用数据库管理，同时有一套后台界面，缺点是相比于Hexo笨重不少……
