数据链路层

  1. 相关技术

    • MAC地址

      MAC地址长48比特(bit)，MAC会烧录到ROM中，任何一个网卡的MAC地址都是唯一的

      实际上，即使MAC地址相同，只要不是同属一个数据链路就不会出现问题

    • 共享介质型网络

      半双工通信

      1. 争用（CSMA/CD协议）

        如果信道上没有数据流动，任何站都可以发送数据。一旦发生冲突，放弃发送数据，立即释放信道。延时一段时间再争用

      2. 令牌传递方式（类似锁）

    • 非共享介质网络

      全双工通信

    • 环路检测技术

      1. 生成树方式
      2. 源路由法

    • VLAN

      在分散网络负载、变换部署网络设备的位置时，不需要重新修改布线，只要修改网段即可。

网络层

  1. IP协议

    1. 基础知识

      • IP地址属于网络层地址
      • 数据链路的地址可以被抽象化为IP地址
      • IP为了简化和提速，采用面向无连接。在发包之前，不需要建立与对端目标之间的连接

    2. IP地址

      IP地址由32位正整数表示，以每8位一组分成4组。

      由网络地址和主机地址组成，现在基本以子网掩码区分

      192.168.1.44/24 前24位为网络地址，后8位为主机地址

      • 网络地址的分类

        • A类

          首位以0开头的地址，1~8位是网络地址。范围在0.0.0.0~127.0.0.0

          一个网络可容纳主机地址上限为2^24-2

        • B类

          前两位为10开头的地址，1~16位是网络地址。范围在128.0.0.0~191.255.0.0

          一个网络可容纳主机地址上限为2^16-2

        • C类

          前三位为110开头的地址，1~24位是网络地址。范围在192.0.0.0~223.255.255.0

          一个网络可容纳主机地址上限为2^8-2 = 254

        • D类

          开头为1110的地址，1~32位是网络地址。范围在224.0.0.0~239.255.255.255

          常用于多播

      • 广播

        在同一个链路中互联的主机之间广播发送数据包，广播地址为IP地址中网络地址部分全部为1

        即在172.20.0.0/16的网络中，广播地址为172.20.255.255

        广播分本地广播和直接广播，不同网络之间的广播叫直接广播

      • IP多播

        多播用于将包发送给特定组内的所有主机

      • 子网掩码

        现在受子网掩码的区分，将子类网络细分出比A类、B类、C类粒度更小的网络。

        它对应IP地址网络地址部分的位全部为“1”，对应主机地址的位全部为“0”

        即172.20.100.0/26对应的子网掩码为255.255.255.192

      • 全局地址与私有地址

        对没有连接互联网的独立网络中的主机，只要保证在本网络中内地址唯一，可以不用考虑互联网即可配置相应的IP地址。即出现了私有网络的IP地址

        • A类私有IP

          10.0.0.0~10.255.255.255 (10.0.0.0/8)

        • B类私有IP

          172.16.0.0 ~ 172.31.255.255 (172.16.0.0/12)

        • C类私有IP

          192.168.0.0~ 192.168.255.255 (192.168.0.0/16)

        在此之外的IP地址称为全局IP。

        如果配有私有IP地址的主机联网时，则通过NAT进行通信

    3. 路由控制

      路由控制表的形成方式有两种：

      1. 管理员手动设置，也叫静态路由控制
      2. 路由器与其他路由器相互交换信息时自动刷新，也叫动态路由控制

      • 默认路由

        默认路由指路由表中任何一个地址都能与之匹配的记录，为0.0.0.0/0

      • 主机路由

        IP地址/32 称为主机路由。意味着基于主机上网卡上配置的IP地址本身，而不是基于该地址的网络地址部分进行路由

      • 环回地址

        使用127.0.0.1作为还回地址，使用这个地址时数据包不会流向网络

    4. IP报文分片

      由于数据链路不同，MTU也不同。IP报文的最大MTU为2^16-1。在目标主机端重组分片IP数据报成为现在的规范，但是对路由器负担加重

      • 路径MTU发现

        路径MTU（Path MTU）指从发送端主机到接收端主机之间不需要分片时最大MTU的大小，即路径中存在的所有数据链路中最小的MTU。

        路径MTU发现从发送主机按照路径MTU的大小将数据包分片后进行发送，可以避免在中途的路由器上进行分片处理

      发送主机与路由器以FDDI连接，MTU = 4352，而以太网MTU=1500

      路由器会丢弃主机第一次发送的大小4352的包，然后通过ICMP的不可达消息将数据链路上MTU的值发给主机（即1500）

      随后主机分片后再以1500的包发给路由器

    5. IPv6

      IPv6地址长为128比特，一般写成8个16位字节，如FEDC:BA98:7642:3215:DCF3:BA98:6723:CB39

      在IPv6中，如果出现连续的0可以将0省略并用两个冒号::隔开，一个IP地址中仅允许出现一次两个连续的冒号

    6. 首部长度

      IPv4首部长度为20字节

  2. ARP

    ARP，地址解析协议（Address Resolution Protocol）

    实际通信时，必须同时知道IP地址和MAC地址（IP地址有层次性而MAC地址没有，仅凭MAC地址无法确定计算机的位置）。ARP借助ARP请求与ARP相应两种类型的包确定MAC地址。

    在网络A中，主机1想知道主机2的MAC地址，要通过广播发送一个ARP请求包，包中包含想要了解MAC地址的IP地址，目标IP与本身IP一致时，主机2将自己的MAC地址塞入ARP响应包返回给A

    为了避免不必要的网络流量，会把获取到的MAC地址缓存一段时间，即把第一次通过ARP获取到的MAC地址作为IP对MAC的映射关系存到一个ARP缓存表中

    • RARP

      Reverse ARP，即通过MAC地址获取IP地址

    • 代理ARP

      通过Proxy ARP将ARP请求转发给临近的网段

  3. ICMP

    网络控制消息协议（Internet Control Message Protocol），用于在IP主机、路由器之间传递网络通不通、主机是否可达、路由是否可用等网络本身的控制信息

    主机A给主机B发送一个ARP请求包，路由1未能发现主机B的存在，于是给主机A发送一个ICMP包，说明发往主机B的包未能成功

    ICMP的通知消息分两类

    1. 通知出错原因的错误信息
    2. 用于诊断的查询信息

    以明文的形式通过IP进行传输

    主要的ICMP消息

    • ICMP目标不可达

    • ICMP重定向消息

      路由器发现发送端主机使用了次优的路径发送数据，它会返回一个ICMP重定向消息给主机，包含最合适的路由信息和源数据。

    • ICMP超时消息

      IP包中有一个字段叫TTL（Time To Live）生存周期，它的值随每经过一个路由器就会减1，直到减到0该IP包就会被丢弃，同时给主机发送一个ICMP超时的消息，通知该包已经被丢弃

    • ICMP回送消息

      判断发送的数据包是否已经成功到达对端的一种消息。ping命令基于此实现

  4. DHCP

    动态主机配置协议（Dynamic Host Configuration Protocol）

    使用前需要先假设一台DHCP服务器，将所要分配的IP地址、相应的子网掩码、路由控制信息、DNS服务器地址设置到服务器上。

    • 工作原理

      1. DHCP客户端广播发送DHCP发现包，要求设置IP地址和子网掩码；

        DHCP服务器接受该包并向客户端返回DHCP提供包，通知可以使用的网络设置

      2. DHCP客户端广播发送DHCP请求包，通知服务器想要使用的设置；

        DHCP服务器接受该包并向客户端返回DHCP提供包，通知允许客户端的设置

      这样保证即使重复设置也能正常工作

    在大规模组织机构的网络环境中，一般会设置DHCP中继代理

  5. NAT

    ​Network Address Translator 网络地址转换，是用于在本地网络中使用私有地址，在连接互联网时专用全局IP地址的技术
