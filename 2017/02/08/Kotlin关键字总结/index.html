Kotlin关键字总结

Kotlin关键字总结

  • val

    只读属性

    ​

  • var

    可变属性

    ​

  • const

    编译时常量，需要满足

    • 以String或基本类型进行初始化
    • 没有自定义getter
    • 对象成员或顶级（Top-level）

    ​

  • lateinit

    延迟初始化属性

    只能被用在类的var可变属性定义中，不能用在构造方法中，属性不能有自定义setter/getter，属性类型非空，也不能为基本类型

    ​

  • field

    备用字段

    ​

  • data

    数据类，编译器会自动添加equals/hashCode/toString()/copy()方法

    • 主构造函数应该至少有一个参数
    • 主构造函数的所有参数必须标注为val或者var
    • 数据类不能是abstract/open/sealed/inner
    • 不能继承其他类（可以实现接口）
    • JVM中如果构造函数是无参的，则所有属性必须有默认的值

    ​

  • open

    允许别的类继承这个类

    默认情况下，kotlin中所有的类都是final的

    ​

  • public

    在任何地方可见

    ​

  • protected

    和private大体相似，但在子类中也可见

    ​

  • private

    只在该类（以及它的成员）中可见

    ​

  • internal

    在本模块的所有可以访问到声明区域的均可以访问该类的所有internal成员

    ​

  • interface

    接口，同Java

    接口的方法可以有自己的实现，可以有属性但必须是抽象的、或提供访问器的实现

    ​

  • abstract

    抽象类，同Java

    ​

  • final

    同Java

    ​

  • override

    复写父类的函数，如果不加则非法

    ​

  • constructor

    一个类可以有一个主构造函数以及多个二级构造函数

    如果主构造函数没有注解或者可见性说明，constructor关键字是可以省略

    ​

  • sealed

    类名关键字，密封类，值只能是有限集合中的某种类型，相当于是一个枚举类的扩展。密封类可以有子类但必须全部嵌套在密封类内部声明

    ​

  • inner

    类可以标记为inner这样就可以访问外部类的成员，拥有外部类的一个对象引用

    ​

  • enum

    枚举类，同Java

    ​

  • companion

    伴生对象，可以使用类名作为修饰符调用伴生对象的成员。伴生对象的名字可以被省略，因为在调用伴生对象成员时不涉及伴生对象的名字，这种情况下在使用伴生对象时用Companion替代伴生对象名

    伴生对象看起来像其它语言中的静态成员，但是在运行时他们仍然是对象的实例成员，并且可以实现接口

    ​

  • object

    Kotlin用对象表达式和对象声明实现Java匿名内部类这种情况，即匿名类的实现对象为object

    通过这种方式，我们可以很轻松地实现单例

    如果object关键字后面有名字，它不再是一个表达式。不能将它赋值给变量，但是我们可以通过他的名字引用它。这样的对象可以拥有超类

    ​

  • @

    表示a为A的引用（内部类中明确引用）

    ​

  • as

    类型转换

    将y转换为String类型

    ​

  • is

    即instance，类型检查

    ​

  • out

    ​

  • ?.

    安全调用

    b?.length，b为空时返回null，否则返回Int?

    ​

  • !.

    b!.length，要求b不能为空，否则编译错误

    ​

  • !!.

    b!!.length，返回一个非空对象(Int?)或者抛出一个NPE

    ​

  • ?;

    左边表达式不为空时返回表达式的结果，否则返回右边的值
