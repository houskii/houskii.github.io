记最近配置环境时碰到的坑

最近一直在看kotlin的东西，然后参考了大帅MarkZhai的从零开始的Android新项目尝试自己按照实际开发的流程去写点东西，结果配置环境的时候碰到了一些坑，在这里记录一下做个总结

  • 坑1：Kotlin+DataBinding

    写demo的时候发现DataBinding无法解析出layout中Kotlin的数据类，但是用Java版本是没问题，一开始怀疑DataBinding不支持Kt，但是问了一下大精Wing，发现他的项目里面是正常使用的没什么问题。然后就只能自己找问题了……

    我当时的grade是这样的

    在网上看到别人告诉我kotlin用的apt是kapt，所以要单独加上，然后我就改了app的gradle

    依然报错，又找了好久，发现在StackOverFlow上有这样一句话，我翻译一下：

    当kotlin和java都要使用apt时，应当仅使用kapt

    好家伙，然而当我把android的apt注销掉，还是报错。。。。。。

    对比了一下别人成功的Kotlin+Databinding配置，我又把'kotlin-android-extensions'改成了'kotlin-android'，最后变成了这样

    ​

    终于可以通过编译了。。。

    ​

  • 坑2：dex cannot parse version 52 byte code

    年轻的我以为这样就结束了，然而并没有。编译确实通过了，但是运行时却爆出这样一段话

    Error:Error converting bytecode to dex:

    Cause: Dex cannot parse version 52 byte code. This is caused by library dependencies that have been compiled using Java 8 or above. If you are using the ‘java’ gradle plugin in a library submodule add targetCompatibility = ‘1.7’ sourceCompatibility = ‘1.7’ to that submodule’s build.gradle file.

    网上找了一圈，大多都是这样的解决方案

    或者

    再看看错误，说是有模块使用了Java8，那么我就干脆把整个工程改成Java8，Android中使用Java8的话就必须要使用Jack&Jill编译器

    还是不行，不过这次错误原因却不同了。在网上贴了错误代码后，在JakeWharton大神的Github中，发现有人提到了同样的问题，而JakeWharton大神的回复是：目前Jack编译器不支持apt 。。。

    实在没办法了，只能去看看哪个包那么贱上了Java8

    找到最后发现，原来是dagger-compiler模块……因为我是直接去github拿的版本号，那么问题就可能是在compiler中使用了lambda或者别的特性，把daggerVersion换成2.0.2后，编译终于通过了，app也可以成功运行了。
